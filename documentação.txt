5. Módulo: state.js
Propósito

Fonte única de verdade. Define todas as variáveis globais, valores padrão e funções de persistência.
Variáveis Definidas

JavaScript

const DEFAULTS = {
    apiKeys: [''],       // Array de chaves API do Gemini
    currentKeyIdx: 0,    // Índice da chave em uso (rotação)
    systemPrompt: '',    // Instrução de sistema
    temperature: 1,      // 0-2, controle de criatividade
    maxTokens: 8192,     // 256-65536, limite de saída
    topP: 0.95,          // 0-1, nucleus sampling
    model: 'gemini-2.5-flash',  // Modelo Gemini
    thinking: true,      // Habilitar chain of thought
    thinkingBudget: 8192,// Tokens para pensamento interno
    streaming: true,     // Resposta em tempo real via SSE
    theme: 'dark'        // Tema visual ativo
};

let S = {...DEFAULTS};   // Config ativa (mutável)
let chats = [];          // Array<Chat>
let activeId = null;     // string | null
let generating = false;  // Semáforo de geração
let aborter = null;      // AbortController | null
let pendingImages = [];  // Array<ImageData> (imagens não enviadas)
let searchFilter = '';   // Filtro de busca na sidebar

Funções

JavaScript

el(id: string): HTMLElement | null
// Atalho para document.getElementById
// Usado por TODOS os módulos

save(): void
// Serializa S, chats e activeId para localStorage
// Chaves: 'rai_s', 'rai_c', 'rai_a'
// Trata QuotaExceededError com toast de aviso
// Chamado após TODA mutação de estado

load(): void
// Deserializa do localStorage para variáveis globais
// Aplica DEFAULTS como fallback para campos ausentes
// Normaliza apiKeys (garante Array)
// Valida activeId contra chats existentes
// Chamado UMA VEZ na inicialização (app.js)

Regras de Mutação

text

✅ Qualquer módulo pode LER S, chats, activeId, generating
✅ Qualquer módulo pode ESCREVER nessas variáveis
✅ Após escrever, DEVE chamar save() para persistir
⚠️ generating deve ser true APENAS durante send/regen
⚠️ aborter deve ser null quando não há geração ativa

Exemplo de Uso em Outro Módulo

JavaScript

// Ler configuração
const model = S.model;
const temp = S.temperature;

// Modificar e persistir
S.temperature = 1.5;
save();

// Acessar elemento DOM
const input = el('inp');

6. Módulo: utils.js
Propósito

Funções utilitárias puras (sem side effects no estado), ícones SVG e registros globais (window.R).
Dependências

    state.js: usa el() para toastsEl

Funções

JavaScript

uid(): string
// Gera ID único: 'c' + timestamp + '_' + random(6)
// Formato: "c1719345678901_a3bx2k"
// Usado para: IDs de chat, IDs únicos de stream

toast(msg: string, icon?: string): void
// Exibe notificação temporária (3 segundos)
// Parâmetros:
//   msg  - Texto da mensagem
//   icon - Emoji/texto do ícone (padrão: 'ℹ️')
// Onde: #toasts (position: fixed, top-right)
// Auto-remove após 3s com animação CSS

esc(s: string): string
// Escapa HTML para prevenir XSS
// Usa textContent → innerHTML trick
// Entrada: '<script>alert("xss")</script>'
// Saída:   '&lt;script&gt;alert(&quot;xss&quot;)&lt;/script&gt;'
// Retorna '' para input falsy

wordCount(text: string): number
// Conta palavras separadas por whitespace
// "Hello world" → 2
// "" → 0, null → 0

md(text: string): string
// Converte Markdown para HTML
// Retorna string HTML segura (conteúdo é escapado)
//
// ═══ PIPELINE DE CONVERSÃO ═══
//
// 1. Protege blocos de código (```lang\n...\n```)
//    → Substitui por %%CODEBLOCK_N%% placeholder
//
// 2. Protege código inline (`...`)
//    → Substitui por %%INLINE_N%% placeholder
//
// 3. Escapa TODO o HTML restante com esc()
//
// 4. Restaura código inline com <code>
//
// 5. Restaura blocos de código com:
//    <div class="code-wrap">
//      <div class="code-head">
//        <span>{lang}</span>
//        <button class="copy-btn" onclick="R.copyEl(this)">Copiar</button>
//      </div>
//      <pre><code>{código escapado}</code></pre>
//    </div>
//
// 6. Aplica formatação inline:
//    **bold**     → <strong>
//    *italic*     → <em>
//    ~~strike~~   → <del>
//    [text](url)  → <a href target="_blank">
//
// 7. Aplica formatação block:
//    # H1, ## H2, ### H3
//    > blockquote
//    --- → <hr>
//    1. item → <ol><li>
//    - item  → <ul><li>
//
// 8. Parágrafos: \n\n → </p><p>
//
// 9. Limpa <p> ao redor de block elements
//
// ═══ ELEMENTOS SUPORTADOS ═══
// ✅ Code blocks com language tag
// ✅ Inline code
// ✅ Bold, italic, strikethrough
// ✅ H1-H3
// ✅ Blockquote
// ✅ Horizontal rule
// ✅ Ordered lists (1. 2. 3.)
// ✅ Unordered lists (- ou *)
// ✅ Links com target="_blank"
// ✅ Parágrafos automáticos
// ❌ Tabelas (CSS existe mas parser não implementa)
// ❌ Imagens em markdown
// ❌ Listas aninhadas
// ❌ Footnotes

Cuidados ao Modificar

text

⚠️ md() é chamado em TEMPO REAL durante streaming
    → Performance é crítica, evitar regex complexos
    → Testar com textos de 10k+ caracteres

⚠️ esc() é a BASE de segurança contra XSS
    → NUNCA inserir conteúdo do usuário sem esc()
    → md() já escapa internamente

⚠️ SVG strings são injetadas via innerHTML
    → Manter sanitizados, sem user input

⚠️ window.R.copyEl é referenciado por onclick="" no HTML
    gerado pelo md() → não renomear sem atualizar md()

7. Módulo: api.js
Propósito

Toda comunicação com a API Google Gemini. Construção de payloads, rotação de chaves, streaming SSE.
Dependências

    state.js: S, aborter, save()
    utils.js: toast()

Funções

JavaScript

getValidKeys(): string[]
// Filtra S.apiKeys: apenas chaves com length >= 30
// Retorna array vazio se nenhuma válida
// Usado por: callAPI(), updBadge(), send()

buildBody(messages: Message[]): Object
// Constrói o payload JSON para a API Gemini
//
// ═══ ESTRUTURA DO RETORNO ═══
// {
//   contents: [
//     { role: "user"|"model", parts: [{text}, {inlineData}...] }
//   ],
//   generationConfig: {
//     temperature: number,
//     maxOutputTokens: number,
//     topP: number,
//     thinkingConfig?: { thinkingBudget: number }  // só modelos 2.5
//   },
//   systemInstruction?: {        // campo nativo da API
//     parts: [{ text: string }]
//   }
// }
//
// ═══ MAPEAMENTO ═══
// Message.role 'user'  → API role 'user'
// Message.role 'model' → API role 'model'
// Message.content      → parts[].text
// Message.images[]     → parts[].inlineData { mimeType, data(base64) }
//
// ═══ CONDIÇÕES ═══
// systemInstruction: só adicionado se S.systemPrompt não vazio
// thinkingConfig: só se S.thinking=true E modelo contém '2.5'
// images: só se Message.images existe e tem items com .data

callAPI(messages: Message[]): Promise<APIResult>
// Faz a chamada à API com rotação automática de chaves
//
// ═══ RETORNO ═══
// { stream: ReadableStream, isStream: true, keyUsed: number }  // streaming
// { data: Object, isStream: false, keyUsed: number }           // normal
//
// ═══ ROTAÇÃO DE CHAVES ═══
// 1. Começa em S.currentKeyIdx
// 2. Se HTTP 429/403 ou erro de quota → tenta próxima chave
// 3. Itera até esgotar todas as chaves válidas
// 4. Atualiza S.currentKeyIdx na chave que funcionou
//
// ═══ URL CONSTRUÍDA ═══
// Base: https://generativelanguage.googleapis.com/v1beta/models/
// Ação: streamGenerateContent (streaming) ou generateContent (normal)
// Query: ?key={apiKey}&alt=sse (SSE só em streaming)
//
// ═══ ERROS ═══
// AbortError → re-throw (para ser tratado em send.js)
// 429/403/quota → retry com próxima chave
// Outros HTTP → throw Error com mensagem da API
// Todas falham → throw "Todas as chaves API falharam"

parseSSE(stream: ReadableStream): AsyncGenerator<Object>
// Parser de Server-Sent Events
//
// ═══ PROTOCOLO SSE ═══
// Formato: "data: {json}\n\n"
// Fim: "data: [DONE]\n\n"
//
// ═══ FUNCIONAMENTO ═══
// 1. Lê chunks do ReadableStream via reader.read()
// 2. Decodifica com TextDecoder (stream: true para chunks parciais)
// 3. Acumula em buffer, split por \n
// 4. Linhas "data: " → parse JSON → yield
// 5. Buffer residual processado quando stream termina
//
// ═══ YIELD ═══
// Cada yield é um objeto da API com:
// {
//   candidates: [{
//     content: {
//       parts: [{ text?, thought? }]
//     }
//   }],
//   usageMetadata?: { promptTokenCount, candidatesTokenCount, thoughtsTokenCount }
// }
//
// ═══ ROBUSTEZ ═══
// - JSON.parse em try/catch (ignora chunks malformados)
// - reader.releaseLock() em finally
// - Buffer residual processado após done=true

Formato da Resposta da API (referência)

JavaScript

// Resposta não-stream:
{
    candidates: [{
        content: {
            parts: [
                { text: "resposta normal" },
                { text: "pensamento", thought: true }  // quando thinking habilitado
            ]
        },
        finishReason: "STOP" | "MAX_TOKENS" | "SAFETY" | ...
    }],
    usageMetadata: {
        promptTokenCount: 150,
        candidatesTokenCount: 500,
        thoughtsTokenCount: 200  // só com thinking
    }
}

// Chunk de stream:
// Mesmo formato, mas parcial (text acumulativo por chunk)

Cuidados ao Modificar

text

⚠️ callAPI() modifica S.currentKeyIdx como side effect
⚠️ aborter é criado aqui mas controlado por send.js (stopGen)
⚠️ parseSSE() é um AsyncGenerator — DEVE ser consumido com for-await
⚠️ Nunca expor chaves API em logs de produção
⚠️ O campo systemInstruction é nativo do Gemini API v1beta
    → Não precisa de hack com mensagens falsas user/model

8. Módulo: chat.js
Propósito

CRUD completo de chats. Criação, exclusão, renomeação, fixação, bifurcação e edição de mensagens.
Dependências

    state.js: chats, activeId, generating, save()
    utils.js: uid(), toast()
    render.js: renderList(), renderMsgs()
    ui.js: closeMobile(), updBtn()
    send.js: send()

Funções

JavaScript

newChat(): Chat
// Cria novo chat e ativa
// 1. Gera Chat com uid(), título "Novo Chat"
// 2. unshift em chats[] (mais recente primeiro)
// 3. Define activeId
// 4. save() + renderList() + renderMsgs()
// 5. Fecha sidebar mobile + foca input
// Retorna o Chat criado

active(): Chat | undefined
// Retorna o chat com id === activeId
// undefined se nenhum ativo ou não encontrado
// Usado extensivamente em render.js e send.js

switchChat(id: string): void
// Troca chat ativo
// 1. activeId = id
// 2. save() + renderList() + renderMsgs()
// 3. closeMobile()

delChat(id: string): void
// Exclui chat por ID
// 1. Remove de chats[]
// 2. Se era o ativo, ativa o primeiro restante (ou null)
// 3. save() + renderList() + renderMsgs()

renChat(id: string): void
// Renomeia via prompt() do browser
// Não faz nada se cancelado ou vazio
// save() + renderList()

pinChat(id: string): void
// Toggle de fixação (c.pinned = !c.pinned)
// save() + renderList() + toast de feedback

forkChat(chatId: string, msgIdx: number): void
// Bifurca chat a partir de uma mensagem específica
// 1. Deep copy das mensagens[0..msgIdx]
// 2. Cria novo chat com título "🔀 " + título original
// 3. Define forkedFrom = chatId original
// 4. Ativa o novo chat
// Caso de uso: explorar resposta alternativa sem perder histórico

autoTitle(chat: Chat): void
// Auto-nomeia chat baseado na primeira mensagem
// Só age se título atual é "Novo Chat"
// Trunca em 38 caracteres + "..."
// Chamado após primeira mensagem do usuário (em send.js)

Funções

JavaScript

send(): Promise<void>
// Fluxo completo de envio de mensagem
//
// ═══ PIPELINE ═══
//
//  ┌──────────────────────────────────────┐
//  │ 1. VALIDAÇÃO                         │
//  │    - Texto vazio E sem imagens? →sair│
//  │    - generating=true? → sair         │
//  │    - Sem API key? → toast + openSet  │
//  └───────────────┬──────────────────────┘
//                  ↓
//  ┌──────────────────────────────────────┐
//  │ 2. PREPARAÇÃO                        │
//  │    - active() ou newChat()           │
//  │    - Criar Message {role,content}    │
//  │    - Anexar pendingImages se houver  │
//  │    - Push em chat.messages           │
//  │    - autoTitle() + save()            │
//  │    - Limpar input + updBtn           │
//  └───────────────┬──────────────────────┘
//                  ↓
//  ┌──────────────────────────────────────┐
//  │ 3. RENDERIZAÇÃO                      │
//  │    - Remover welcome se presente     │
//  │    - addMsgDOM(userMsg)              │
//  │    - Criar typing indicator (#typInd)│
//  │    - scrollDown()                    │
//  └───────────────┬──────────────────────┘
//                  ↓
//  ┌──────────────────────────────────────┐
//  │ 4. API CALL                          │
//  │    - generating = true               │
//  │    - t0 = Date.now()                 │
//  │    - callAPI(chat.messages)          │
//  │    - Remove typing indicator         │
//  └───────────┬───────────┬──────────────┘
//              ↓           ↓
//     ┌────────────┐ ┌─────────────┐
//     │ isStream   │ │ !isStream   │
//     │ →handle    │ │ →handleFull │
//     │  Stream()  │ │  ()         │
//     └────────────┘ └─────────────┘
//              ↓           ↓
//  ┌──────────────────────────────────────┐
//  │ 5. CLEANUP (finally)                 │
//  │    - generating = false              │
//  │    - aborter = null                  │
//  │    - updBtn() + save()               │
//  │    - renderList() + scrollDown()     │
//  └──────────────────────────────────────┘
//
// ═══ TRATAMENTO DE ERROS ═══
// AbortError → toast "Cancelado" (sem mensagem de erro no chat)
// Outros     → Cria Message de erro no chat + toast

handleFull(data: Object, chat: Chat, t0: number, keyUsed: number): void
// Processa resposta não-stream da API
// 1. Extrai candidates[0]
// 2. Separa parts: thought=true → thinking, else → content
// 3. Extrai usageMetadata
// 4. Cria Message com meta completa
// 5. Push em chat.messages + addMsgDOM + scrollDown

handleStream(stream: ReadableStream, chat: Chat, t0: number, keyUsed: number): Promise<void>
// Processa resposta em streaming
//
// ═══ DOM CRIADO ═══
// Cria .msg com IDs únicos (uid-based para evitar conflito):
//   stThink_{sid}   → think box (display:none inicialmente)
//   stThinkH_{sid}  → think header (toggle click)
//   stThinkB_{sid}  → think body (innerHTML atualizado a cada chunk)
//   stThinkAr_{sid} → think arrow (toggle open)
//   stTxt_{sid}     → texto principal (.streaming class = cursor animado)
//   stMeta_{sid}    → metadados (display:none até final)
//
// ═══ LOOP DE STREAMING ═══
// for await (chunk of parseSSE(stream)):
//   - parts com thought=true → acumula think, atualiza stThinkB
//   - parts sem thought      → acumula txt, atualiza stTxt
//   - usageMetadata          → guarda para meta final
//   - scrollDown() a cada chunk
//
// ═══ FINALIZAÇÃO ═══
// 1. Remove .streaming class (para cursor de digitação)
// 2. Preenche stMeta com tokens/tempo/modelo
// 3. Adiciona word count
// 4. Cria botões de ação (copy, regen)
// 5. Push Message final em chat.messages

stopGen(): void
// Cancela geração em andamento
// Chama aborter.abort() se aborter não é null
// O AbortError é capturado em send() e handleStream()

regen(): Promise<void>
// Regenera última resposta do modelo
//
// ═══ PIPELINE ═══
// 1. Valida: chat existe, >= 2 msgs, !generating
// 2. Encontra último message com role='model' (busca reversa)
// 3. Remove essa mensagem de chat.messages (splice)
// 4. save() + renderMsgs() (re-renderiza sem a resposta)
// 5. Cria typing indicator
// 6. callAPI(chat.messages) — que agora termina na msg do user
// 7. handleStream/handleFull com a nova resposta
// 8. Cleanup igual ao send()
//
// ═══ DIFERENÇA DO send() ═══
// - NÃO cria nova mensagem de usuário
// - NÃO usa input (o texto do user já está no histórico)
// - Remove apenas a resposta do modelo

Fluxo de Estado durante Geração

text

IDLE:     generating=false, aborter=null, btnSend=▶ (send icon)
GERANDO:  generating=true,  aborter=AbortController, btnSend=⏹ (stop)
CANCELAR: aborter.abort() → AbortError → IDLE
ERRO:     mensagem de erro no chat → IDLE
SUCESSO:  resposta no chat → IDLE

Cuidados ao Modificar

text

⚠️ send() é async — o semáforo `generating` previne chamadas paralelas
⚠️ handleStream cria IDs únicos com uid() — NUNCA usar IDs fixos
⚠️ O typing indicator (#typInd) DEVE ser removido em TODOS os caminhos
    (sucesso, erro, abort) — usar document.getElementById para segurança
⚠️ regen() NÃO usa o input — lê direto do chat.messages
⚠️ pendingImages é consumido (limpo) durante send — uma vez enviado, some
⚠️ scrollDown() é chamado a cada chunk de stream — pode causar jank se
    a renderização de md() for lenta

text

⚠️ btnSend usa APENAS addEventListener (não .onclick)

⚠️ O listener do Enter no input verifica !e.shiftKey
    → Shift+Enter = nova linha (comportamento esperado)

    → Botão sempre chama o mesmo handler, que verifica `generating`

⚠️ drag/drop listener está no document, não no input
    → Qualquer lugar da página aceita drop de imagens

⚠️ paste listener está no #inp
    → Só intercepta se for imagem, texto passa normal

interface Chat {

Message

TypeScript

interface Message {
    role: 'user' | 'model';
    content: string;          // Texto da mensagem
    thinking?: string | null; // Conteúdo do chain-of-thought
    images?: ImageData[];     // Imagens anexadas (só user)
    meta?: {                  // Metadados (só model)
        inTok?: number;       // Tokens de entrada
        outTok?: number;      // Tokens de saída
        thinkTok?: number;    // Tokens de pensamento
        dur?: string;         // Duração em segundos "2.3"
        model?: string;       // "gemini-2.5-flash"
        keyUsed?: number;     // Índice da chave API usada
    };
}

18. Fluxos Críticos
18.1 Fluxo: Enviar Mensagem com Streaming

text

Usuário digita + clica enviar
        │
        ▼
    send() [send.js]
        │
        ├─ Valida (texto/imagens, !generating, API key)
        ├─ Cria Message {role:'user', content, images?}
        ├─ Push em chat.messages
        ├─ addMsgDOM(userMsg) [render.js]
        ├─ Cria #typInd (typing dots)
        ├─ generating = true
        │
        ▼
    callAPI(messages) [api.js]
        │
        ├─ buildBody(messages) → JSON payload
        ├─ Loop de chaves (rotação se 429/403)
        ├─ fetch() com AbortController
        │
        ▼
    {stream, isStream:true, keyUsed}
        │
        ├─ Remove #typInd
        │
        ▼
    handleStream(stream) [send.js]
        │
        ├─ Cria DOM: .msg com stTxt_{uid}, stThink_{uid}, etc.
        │
        ▼
    for await (chunk of parseSSE(stream)) [api.js]
        │
        ├─ parts com thought → acumula think, innerHTML stThinkB
        ├─ parts sem thought → acumula txt, innerHTML stTxt
        ├─ scrollDown() cada chunk
        │
        ▼
    Finalização
        │
        ├─ Remove .streaming class
        ├─ Preenche stMeta (tokens, tempo)
        ├─ Adiciona word count + action buttons
        ├─ Push Message completa em chat.messages
        ├─ generating = false
        ├─ save() + renderList()
        └─ scrollDown()

text

        │
        ▼
    ┌─ Tenta chave #N ─────────────────┐
    │  fetch(url?key=chaves[N])        │
    │       │                           │
    │  ┌────┴─────┐                     │

    │  │ Salva N  │   keyUsed = N       │

    │  │ quota..."     │                │
    │  └───┬───────────┘                │
    │      │ tried++                    │

    │      └────────────────────────────┘
    │
    │  tried >= total?
    │      │
    │      ▼
    │  throw "Todas falharam"

Regras Gerais de Contribuição

text

1. NUNCA usar .onclick — sempre addEventListener
2. NUNCA inserir user content sem esc()
3. SEMPRE chamar save() após mutar S, chats ou activeId
4. SEMPRE usar IDs únicos (uid()) para elementos dinâmicos
5. SEMPRE tratar AbortError separadamente de outros erros
6. SEMPRE limpar #typInd em todos os caminhos (success/error/abort)
7. NUNCA adicionar dependências externas
8. NUNCA expor chaves API em console.log
9. Para novos temas: só CSS + HTML (zero JS)
10. Para novos modelos: só HTML no <select> (zero JS/CSS)
